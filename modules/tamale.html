matcher (spec)
Return a matcher function for a given specification.

Parameters

  • spec: A list of rows, where each row is of the form { pattern, result, [where=capture_test_fun(cs)] }. Each table pattern is indexed by pattern[1].

Usage

  • spec.fail: The spec can have an optional function to call when nothing matches. By default, match_fail is used.
  • spec.ids: An optional list of table values that should be compared by identity, not structure. If any empty tables are being used as a sentinel value (e.g. "MAGIC_ID = {}"), list them here.
  • spec.debug: Turn on debugging traces for the matcher.
